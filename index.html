Graph Database Cheat Sheet

Quick reference for everything you've learned — and more.

Core Concepts

Node (Vertex)
An entity in the graph — a person, product, place, event. Has properties (key-value pairs) and labels (types).
Edge (Relationship)
A connection between two nodes. Always directed, always typed (e.g., KNOWS, PURCHASED). Can have properties.
Label
A tag on a node that groups it by type. A node can have multiple labels (e.g., :Person:Employee).
Property
Key-value data stored on nodes or edges. E.g., name: "Alice", since: 2020.
Traversal
Walking the graph by following relationships from node to node. The core operation of graph queries.
Path
A sequence of nodes and edges connecting two nodes. Can be shortest, all, or variable-length.

When to Use a Graph DB

✓ Great For

  • Social networks & relationship queries
  • Recommendation engines
  • Fraud detection & anti-money laundering
  • Knowledge graphs & semantic search
  • Network & IT infrastructure mapping
  • Supply chain & logistics optimization
  • Identity & access management (IAM)
  • Real-time path finding & routing

✗ Not Ideal For

  • Simple CRUD with flat records
  • Heavy aggregation / analytics (OLAP)
  • Binary/blob storage
  • Time-series data (use TimescaleDB, InfluxDB)
  • Full-text search as primary use (use Elasticsearch)
  • High-volume transactional writes with no relationships

Cypher Query Language (Neo4j)

Find nodes

MATCH (p:Person)
WHERE p.name = "Alice"
RETURN p

Find relationships

MATCH (a:Person)-[:KNOWS]->(b:Person)
RETURN a.name, b.name

Variable-length paths

MATCH path = (a)-[:KNOWS*1..3]->(b)
WHERE a.name = "Alice"
RETURN path

Create data

CREATE (a:Person {name: "Alice"})
CREATE (b:Person {name: "Bob"})
CREATE (a)-[:KNOWS {since: 2020}]->(b)

Aggregation

MATCH (p:Person)-[:KNOWS]->(friend)
RETURN p.name, count(friend) AS friends
ORDER BY friends DESC

Shortest path

MATCH path = shortestPath(
  (a:Person {name:"Alice"})
  -[:KNOWS*]->
  (b:Person {name:"Eve"})
)
RETURN path

Graph Data Models

Property Graph (LPG)
Nodes + labeled, directed edges, both with properties. Used by Neo4j, Neptune (via openCypher/Gremlin), TigerGraph. Most common model.
RDF (Resource Description Framework)
Triple-based: subject → predicate → object. Used for semantic web, linked data. Queried with SPARQL. Used by Neptune, GraphDB, Stardog.

Query Languages

Cypher
Declarative, pattern-matching language by Neo4j. ASCII-art syntax: (a)-[:REL]->(b). Most popular. Now an open standard (openCypher / GQL).
Gremlin
Imperative traversal language from Apache TinkerPop. g.V().has('name','Alice').out('KNOWS'). Used by Neptune, Cosmos DB, JanusGraph.
SPARQL
W3C standard for querying RDF graphs. SELECT ?name WHERE { ?p a :Person ; :name ?name }. Used by Neptune (RDF mode), Stardog.
GQL
ISO standard (2024) graph query language. Inspired by Cypher, SQL, and PGQL. Future of the space.

Managed Graph Database Services

AWS

Amazon Neptune

Fully managed graph DB supporting both property graph (Gremlin, openCypher) and RDF (SPARQL). Serverless option available. Deep AWS integration (IAM, VPC, CloudWatch).

Azure

Azure Cosmos DB (Gremlin API)

Multi-model database with a Gremlin-compatible graph API. Global distribution, SLA-backed. Good when you need graph alongside document/key-value in one service.

GCP

Neo4j Aura (via GCP Marketplace)

Google Cloud does not offer a native first-party graph database. Neo4j Aura is available via the GCP Marketplace for managed Cypher-based graphs. Spanner and BigQuery have some graph query features.

Neo4j

Neo4j Aura

Fully managed Neo4j as a service. Available on AWS, GCP, and Azure marketplaces. Free tier available. Native Cypher, APOC procedures, Graph Data Science library.

TigerGraph

TigerGraph Cloud

Managed distributed graph analytics platform. GSQL query language. Strong in deep-link analytics, real-time graph analytics at scale. Available on AWS, Azure, GCP.

Others

Other Notable Options

ArangoDB Cloud (Oasis) — multi-model (graph + document + key-value). Dgraph Cloud — GraphQL-native graph DB. Memgraph — in-memory, Cypher-compatible, streaming-focused. JanusGraph — open-source, Gremlin-based, runs on Cassandra/HBase.

Key Graph Algorithms

Shortest Path (Dijkstra)
Find minimum-cost path between nodes. Used in routing, network analysis.
PageRank
Rank node importance by link structure. Used in search, influence scoring.
Community Detection (Louvain)
Find clusters/groups in graphs. Used in social network analysis, market segmentation.
Betweenness Centrality
Identify bridge nodes that connect communities. Used in network resilience, influence analysis.
Node Similarity
Measure how similar two nodes are based on neighborhoods. Used in recommendations.
